# Valentine Azolibe — Full-Stack Developer Portfolio

Welcome to the repository for my personal portfolio website! I am Valentine Azolibe, a Full-Stack Developer based in Nigeria. This portfolio is designed to showcase my projects, skills, and experience, while providing a seamless, fast, and interactive user experience.

## 🌟 Features

- **Modern Tech Stack**: Built with React and Vite for blazing-fast development and optimized production builds.
- **Responsive Design**: Fully mobile-responsive layouts that look great on any device, achieved using Tailwind CSS v4.
- **Smooth Animations**: Engaging and performant animations powered by Framer Motion.
- **Client-Side Routing**: Fast and seamless page transitions using React Router DOM.
- **Clean Codebase**: Strictly linted with ESLint and structured for maintainability and scalability.

## 🛠️ Tech Stack

- **Frontend Framework**: [React 19](https://react.dev/)
- **Build Tool**: [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/)
- **Animations**: [Framer Motion](https://www.framer.com/motion/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Routing**: [React Router](https://reactrouter.com/)

## 🚀 Getting Started

Follow these instructions to get a copy of the project up and running on your local machine for development and testing purposes.

### Prerequisites

Make sure you have the following installed:
- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- npm (comes with Node.js) or your preferred package manager (yarn, pnpm, bun)

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Mexican1922/portfolio.git
   cd portfolio
   ```
   *(Note: Replace the URL with your actual repository URL if different)*

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Start the development server:**
   ```bash
   npm run dev
   ```

4. **View the site:**
   Open your browser and navigate to `http://localhost:5173` (or the URL provided in your terminal).

## 🏗️ Building for Production

To create an optimized production build, run:

```bash
npm run build
```

This command bundles the app into the `dist` directory, ready to be deployed to any static hosting service (like Vercel, Netlify, or GitHub Pages).

To preview the production build locally:

```bash
npm run preview
```

## 🧹 Linting

This project uses ESLint to enforce code quality and formatting standards. To check the codebase for issues, run:

```bash
npm run lint
```

## 📂 Project Structure

A quick overview of the essential files and directories in this repository:

```
.
├── public/             # Static assets that bypass Vite's build system
├── src/                # Main source code
│   ├── assets/         # Images, fonts, and other compiled assets
│   ├── components/     # Reusable React components (Navbar, Hero, Footer, etc.)
│   ├── context/        # React Context providers (e.g., ThemeContext)
│   ├── data/           # Static data files (e.g., project lists, skills)
│   ├── pages/          # Page-level components (HomePage, AboutPage, etc.)
│   ├── App.jsx         # Root application component and routing setup
│   ├── main.jsx        # Entry point for React
│   └── index.css       # Global styles and Tailwind configuration
├── index.html          # Main HTML template
├── package.json        # Project metadata and dependencies
├── vite.config.js      # Vite configuration file
└── eslint.config.js    # ESLint configuration
```
